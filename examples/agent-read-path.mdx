An agent should treat these docs as an interface boundary: inspect the public model, map the visible shape, and stop before internal authority or live writes. The default traversal follows.

## Default path

<Steps>
  <Step title="Read the overview">
    Read the [Overview](/start/overview) to understand the system model.
  </Step>
  <Step title="Choose a surface">
    Choose a surface in [Integration surfaces](/start/integration-surfaces).
  </Step>
  <Step title="Inspect states and actors">
    Inspect lifecycle states in [Public states](/workflow/public-states) and who acts in [Actor boundaries](/workflow/actor-boundaries).
  </Step>
  <Step title="Inspect record shapes">
    Inspect the input, artifact, status, release, and receipt shapes in [Records](/records/input-record).
  </Step>
  <Step title="Map a synthetic flow">
    Map a synthetic flow using the other pages in this section.
  </Step>
  <Step title="Stop before writes">
    Stop before live writes, approval, release, or internal review reasoning.
  </Step>
</Steps>

## Safe outputs

| Output | Safe when |
| --- | --- |
| State explanation | It only describes public lifecycle meaning. |
| Data mapping | It maps fields without sending live data. |
| Shape validation | It checks required structure, not professional criteria. |
| Integration plan | It names current and future surfaces honestly. |

## Stop conditions

<Warning>
Stop if the task asks the agent to infer approval criteria, reconstruct internal methodology, mutate official state, release a deliverable, or expose private workpapers. The full rules are in the [Agent guide](/start/agent-guide).
</Warning>
