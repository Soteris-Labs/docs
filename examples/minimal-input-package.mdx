A minimal input record shows how an external system can describe submitted material without assuming a live write endpoint.

Use this as a shape reference for mapping, validation, and agent context. It is not a production API request.

## Example

```json
{
  "schemaVersion": "2026-07-v0",
  "engagementId": "eng_synthetic_001",
  "source": {
    "type": "partner_system",
    "name": "Example CRM"
  },
  "submittedAt": "2026-07-06T14:30:00Z",
  "documents": [
    {
      "documentId": "doc_synthetic_001",
      "label": "Trial balance export",
      "mimeType": "text/csv",
      "period": "2026-Q2"
    }
  ],
  "notes": "Synthetic example for docs only.",
  "metadata": {
    "clientReference": "client_demo",
    "serviceLine": "crypto_assurance"
  }
}
```

## What to inspect

| Field | Purpose |
| --- | --- |
| `schemaVersion` | Identifies the documented shape. |
| `engagementId` | Connects the record to a Soteris engagement. |
| `source` | Names where the record came from. |
| `documents` | Lists submitted material without exposing file contents. |
| `metadata` | Carries partner-safe mapping context. |

## Boundary

<Info>
  This example describes shape only. It does not grant write access, upload access, or review approval.
</Info>
