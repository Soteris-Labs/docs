---
title: "Release receipt"
description: "A synthetic receipt for a released artifact."
tag: "Public"
---

A release receipt records that a specific artifact version was authorized for a defined audience at a defined time.

Receipts support inspection. They do not disclose private review reasoning.

## Example

```json
{
  "receiptId": "rcpt_synthetic_001",
  "engagementId": "eng_synthetic_001",
  "artifactId": "art_synthetic_001",
  "releaseVersion": "1.0.0",
  "issuedAt": "2026-07-06T16:00:00Z",
  "visibility": "permissioned",
  "status": "active",
  "verificationUrl": "https://docs.soterislabs.xyz/verify/rcpt_synthetic_001"
}
```

## What it proves

| Question | Answered by the receipt |
| --- | --- |
| Was this artifact released? | Yes, if the receipt is active. |
| Which version was released? | `releaseVersion`. |
| When was it released? | `issuedAt`. |
| Who can inspect it? | `visibility`. |

## What it does not prove

<Info>
  The receipt does not reveal internal evidence, reviewer notes, scoring, prompts, or professional judgment rationale.
</Info>
